# 可研报告智能审查助手 Windows 桌面版

本项目当前已调整为 Windows 桌面软件方向，主要面向本地化、离线化、涉密场景下的可研报告智能审查。

用户侧不再使用浏览器访问网页，而是通过 Windows 桌面客户端完成登录、报告选择、检测项选择和审查任务创建。后续后端服务将以本地服务形式接入，默认保证可研报告原文不上传云端。

## 项目定位

本系统用于辅助审查可研报告中的关键内容，当前前端重点展示九项审查能力入口：

1. 建设依据审查
2. 重复建设检查
3. 建设功能的对应关系检查
4. 数据填报合理性
5. 资源申请的合理性
6. 安全内容的合理性
7. 价格合理性
8. 软硬件产品价格参考
9. 非信创内容检查

当前阶段已完成 Windows 桌面客户端原型页面，后端具体审查逻辑后续继续开发。


```bash
dotnet run --project desktop-wpf\AiReportDesktop.csproj
```

```bash

```

## 后端规划

后端建议采用本地服务方式运行，不部署到云端。推荐结构：

```text
WPF 桌面客户端
  ↓
本地后端服务
  ↓
SQLite / 本地文件 / 本地知识库 / 本地规则 / 本地模型
```

后端职责：

- 接收本地报告文件
- 解析 Word、PDF、Excel 等材料
- 抽取九项审查所需的必要信息
- 执行规则审查或本地模型审查
- 保存检测任务、检测结果和审查记录
- 向 WPF 前端返回任务状态和结果

## 前后端连接方式

后续前端和后端建议通过本机接口连接：

```text
http://127.0.0.1:端口号
```

典型流程：

```text
1. 用户打开 WPF 桌面软件
2. 用户选择可研报告文件
3. WPF 将文件提交给本地后端服务
4. 后端在本地解析报告
5. 用户选择九项检测中的部分或全部
6. 后端创建检测任务
7. WPF 查询任务状态
8. 后端返回检测结果
9. WPF 展示结果并支持后续导出
```

## 数据安全设计

项目面向离线或涉密场景，默认原则是：

```text
可研报告原文不上传云端
```

建议后续实现时遵循：

- 报告原文只保存在本机或客户内网指定目录
- 本地完成文档解析和字段抽取
- 检测记录写入本地 SQLite
- 敏感字段支持脱敏处理
- 如需联网 AI 能力，必须提供明确开关和用户确认
