# 可研报告智能审查助手 Windows 桌面版

本项目是面向可研报告审查场景的 Windows 桌面客户端与服务器端后端服务组合。用户侧通过 Windows 桌面客户端完成登录、报告选择、检测项选择和审查任务创建；后端服务和数据库部署在服务器上，统一负责数据存储、任务处理、审查逻辑和接口返回。

## 项目定位

本系统用于辅助审查可研报告中的关键内容，当前重点覆盖九项审查能力入口：

1. 建设依据审查
2. 重复建设检查
3. 建设功能的对应关系检查
4. 数据填报合理性
5. 资源申请的合理性
6. 安全内容的合理性
7. 价格合理性
8. 软硬件产品价格参考
9. 非信创内容检查

当前阶段已完成 Windows 桌面客户端原型页面，并保留 FastAPI 后端服务目录。后续开发应围绕“桌面客户端调用服务器后端接口”的模式继续推进。

## 总体架构

```text
Windows WPF 桌面客户端
  -> HTTPS / HTTP 内网接口
服务器端 FastAPI 后端服务
  ->
服务器端数据库 / 文件存储 / 知识库 / 规则库 / 模型服务
```

说明：

- 桌面客户端安装在用户电脑上。
- 后端服务部署在服务器上，而不是部署在用户本机。
- 数据库部署在服务器上，而不是使用用户本机数据库作为正式数据源。
- 客户端通过服务器接口提交审查任务、查询任务状态、获取审查结果。
- 开发环境可以临时使用 SQLite 进行调试，但生产环境应切换为服务器数据库。

```text

```

技术栈：

```text
Python
FastAPI

```

后端应部署在服务器上，主要职责包括：

- 提供客户端登录、项目管理、审查任务创建等接口
- 接收客户端提交的报告文件或结构化数据
- 解析 Word、PDF、Excel 等材料
- 抽取九项审查所需的必要信息
- 执行规则审查、相似度审查或模型审查
- 保存用户、项目、任务、审查结果和审查记录
- 向 WPF 客户端返回任务状态和审查结果

## 数据库说明

正式环境中，数据库应部署在服务器上。建议使用：

```text
MySQL / PostgreSQL / SQL Server
```

当前 `.env.example` 中的 SQLite 配置仅适合本地开发调试：

```text
DATABASE_URL=sqlite:///./data/app.db
```

服务器部署时应改为服务器数据库连接，例如：

```text
DATABASE_URL=mysql+pymysql://用户名:密码@数据库服务器地址:3306/数据库名
```

或：

```text
DATABASE_URL=postgresql+psycopg://用户名:密码@数据库服务器地址:5432/数据库名
```

真实的数据库账号、密码、密钥等内容不得提交到 GitHub，应放在服务器环境变量或服务器 `.env` 文件中。

## 启动桌面客户端

在项目根目录双击：

```bat
run_desktop.bat
```

或在命令行运行：

```powershell
dotnet run --project desktop-wpf\AiReportDesktop.csproj
```

## 构建桌面客户端

```powershell
dotnet build desktop-wpf\AiReportDesktop.csproj
```

## 本地开发后端

进入后端目录：

```powershell
cd backend
```

安装依赖：

```powershell
pip install -r requirements.txt
```

启动开发服务：

```powershell
uvicorn app.main:app --reload
```

注意：本地启动后端只用于开发调试。正式使用时，桌面客户端应连接服务器上的后端接口。

## 前后端连接方式

生产环境中，WPF 客户端应连接服务器接口，例如：

```text
https://服务器域名或服务器IP/api
```

典型流程：

```text
1. 用户打开 WPF 桌面软件
2. 用户登录服务器端账号
3. 用户选择可研报告文件
4. WPF 将任务请求提交到服务器后端
5. 服务器后端保存任务和必要文件
6. 服务器后端执行报告解析和审查逻辑
7. WPF 查询服务器端任务状态
8. 服务器返回审查结果
9. WPF 展示结果并支持后续导出
```

## 数据安全设计

项目应根据实际部署环境决定数据安全策略。推荐原则：

- 后端服务器和数据库部署在单位内网或受控服务器环境
- 报告原文、审查结果和任务记录存储在服务器端
- 数据库账号、接口密钥、模型密钥不进入 Git 仓库
- 客户端与服务器之间建议使用 HTTPS 或内网安全通道
- 敏感字段支持脱敏处理
- 服务器应做好权限控制、访问审计和备份策略
